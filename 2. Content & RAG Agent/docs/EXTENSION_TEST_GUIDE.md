# 🧩 크롬 확장 프로그램 & 데모 통합 기능 테스트 가이드

이 가이드는 로컬 환경에서 **크롬 확장 프로그램(Chrome Extension)**을 로드하고, 백엔드와 연동하여 모든 핵심 기능(실시간 단어 Hover 사전, OX 퀴즈 동적 개입, 난이도 대시보드 반영)이 정상 작동하는지 수동으로 실시간 테스트할 수 있는 매뉴얼입니다.

---

## 1. 사전 자동화 API 테스트 결과 (Preflight OK)
로컬에서 크롬 확장 프로그램이 사용하는 백엔드 API 규격과 세션 이벤트를 시뮬레이션하는 단위 테스트 **14개 항목 모두 100% 그린(PASSED) 통과** 완료하였습니다. 
백엔드 API(이벤트 로깅, 세션 생성, CORS 헤더 제공 등)는 완벽하게 무결한 상태입니다.

---

## 2. [Step 1] 로컬 통합 데모 서버 실행

백엔드 서버와 프론트엔드가 싱글 포트(`8000`)에서 유기적으로 통합되어 돌아가는 데모 서버를 먼저 가동합니다.

1. **파워쉘(PowerShell) 창**을 엽니다.
2. 아래 명령어로 데모 기동 스크립트를 실행합니다. (1~2분 소요)
   ```powershell
   cd "C:\Users\haoj9\Desktop\Project\AI-literacy-care-Agent"
   .\demo\start_demo.ps1
   ```
   * *기능*: 프론트 빌드(npm install 및 build) → 백엔드 DB 자동 시딩 → `localhost:8000` 로컬 웹서버 기동 및 브라우저 자동 오픈.
   * *종료 시*: 데모를 끄려면 `.\demo\stop_demo.bat`을 실행하면 됩니다.

---

## 3. [Step 2] 크롬 브라우저에 확장 프로그램 로드

1. 크롬(Chrome) 브라우저를 열고 주소창에 `chrome://extensions/` 을 입력하여 이동합니다.
2. 우측 상단의 **"개발자 모드 (Developer mode)"** 스위치를 **ON(활성화)**으로 켭니다.
3. 좌측 상단의 **"압축해제된 확장 프로그램을 로드합니다 (Load unpacked)"** 버튼을 클릭합니다.
4. 아래의 로컬 확장 프로그램 폴더를 선택하여 업로드합니다.
   * 📁 **선택할 폴더 경로**: `C:\Users\haoj9\Desktop\Project\AI-literacy-care-Agent\1. Agent Core & Orchestration\extension`
5. 업로드가 완료되면 크롬 툴바 우측의 **퍼즐 조각 아이콘(확장 프로그램 목록)**을 눌러 **"AI Literacy Care"** 확장 프로그램을 핀(Pin) 고정해 둡니다.

---

## 4. [Step 3] 핵심 기능 시나리오 테스트 진행

데모 브라우저(`http://localhost:8000`)에 접속하여 로그인 및 읽기 환경을 시작한 뒤 아래 시나리오를 하나씩 수행하며 기능 작동을 테스트합니다.

### 🧪 시나리오 A: 실시간 드래그 단어 검색 (2번 RAG 사전 연동)
1. 읽기 모드 화면에서 본문 속 아무 한글 단어(예: "양자역학" 혹은 일반 단어)를 마우스로 **드래그(Drag)**합니다.
2. 드래그하는 순간 화면 우측에 **용어 사전 팝업(Tooltip/Sidebar)**이 즉시 나타나며, RAG 용어풀이 사전에 등록된 표준 용어 정의가 딜레이 없이 팝업되는지 확인합니다.
   * *원리*: 크롬의 `content_script.js`가 드래그 이벤트를 감지하여 백엔드 `/api/terms/lookup`으로 호출하고, 2번 RAG 사전 엔진이 캐시 사전에서 0ms 수준으로 검색하여 돌려줍니다.

### 🧪 시나리오 B: 집중도 감점 및 동적 O/X 퀴즈 개입 (2번 문단 요약 연동)
1. 본문을 정상적으로 읽지 않고, 마우스 휠을 아래위로 극단적으로 **빠르게 굴리거나(Fast Scroll)**, 브라우저 창 밖으로 마우스 포커스를 빼서 **화면을 이탈(Blur)**합니다.
2. 로컬 화면 우측 라이브 모니터에서 **집중도 점수(Focus Score)가 감점(60점 이하로 급락)**되는지 확인합니다.
3. 점수가 급락하는 순간 백엔드 WebSocket 신호에 의해 화면에 **"잠시 집중해 볼까요? 퀴즈 타임!"** 이라는 **O/X 퀴즈 모달 팝업**이 강제로 트리거되는지 확인합니다.
4. 이때 출제된 O/X 퀴즈의 문맥이, 2번이 생성한 **독자 문해력 레벨 맞춤형 요약문(summary)**을 정확히 소스로 삼아 깔끔하고 매끄러운 1문장 퀴즈로 출력되는지 문장 품질을 확인합니다.

### 🧪 시나리오 C: 최종 문해력 성적 및 지수 난이도 대시보드 반영
1. 글을 끝까지 읽은 뒤 우측 하단의 **"읽기 완료 (Finish)"** 버튼을 누릅니다.
2. 성장 대시보드로 리다이렉트되며 최종 성적 분석 결과를 확인합니다.
3. 이때 성적표에 표시되는 글의 난이도가 우리가 개편한 **비선형 지수 난이도 공식**에 따라, 전문 용어가 단 하나라도 존재했다면 높은 체감 난이도(예: 70~98점대)로 정확히 표시되는지 검증합니다.
