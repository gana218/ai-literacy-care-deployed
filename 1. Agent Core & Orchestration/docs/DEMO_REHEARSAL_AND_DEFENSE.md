## 0. 한 문장 정의 (오프닝·클로징에 반복)

> **"AI가 글을 대신 읽어주는 시대에, 우리는 요약을 하나 더 만드는 게 아니라 — 사용자의 *읽는 힘*을 측정하고, 실시간으로 개입하고, 설명 가능한 점수로 되돌려주는 폐루프 케어 시스템입니다."**

핵심 3줄(심사가 기억할 것):
1. **대체가 아니라 회복** — 글을 대신 읽는 게 아니라, 사람이 읽는 힘을 되찾게 한다.
2. **재현 가능한 평가** — 점수를 LLM에 맡기지 않는다. 같은 입력 = 같은 점수(순수 함수 + 근거).
3. **환각 없는 도움** — 용어 뜻은 생성이 아니라 신뢰 사전 **검색**.

## 1. 데모 리허설 스크립트 (약 3분)

**준비물**: 백엔드 실행(`uvicorn backend.app.main:app`), 확장 로드(또는 무설치 웹 폴백), 데모 기사 1편.

| # | 화면/조작 | 멘트(요지) | 뒤에서 실제로 일어나는 일 |
|---|---|---|---|
| 1 | 확장 팝업 ON | "설치·로그인 없이, 토글 하나로 시작합니다. **익명 UUID**만 쓰고 개인정보는 없어요." | `storage.enabled=true` → content script arm, `getUserId()`가 설치별 UUID 발급 |
| 2 | 기사 페이지 진입, 3초 체류 | "보던 대로 읽기만 하면 세션이 자동 시작됩니다." | `POST /session/start` content[] → **청크 4개 + 용어 4개**(인공지능·문해력·환각·알고리즘) |
| 3 | 우하단 **집중도 배지** | "지금부터 읽기 행동을 실시간 측정합니다. 우하단이 실시간 집중도예요." | scroll/blur/pause 이벤트 → `POST /events` → `focusScore` |
| 4 | 딴짓(탭 이탈) 몇 번 | "집중이 흐트러지면 — 단계적으로 개입합니다. 처음엔 약하게." | focus **78→56→34→12**, 개입 **없음→하이라이트(soft)→넛지(medium)→퀴즈(hard)** |
| 5 | **개입 오버레이** 토스트 | "집중도 임계 **75/50/30**에서 단계가 올라가고, 회복하면 개입은 알아서 사라집니다(과잉 개입 방지)." | routing.py가 임계 75/50/30로 개입 결정, overlay 토스트 렌더 |
| 6 | (선택) 어려운 단어 hover | "어려운 용어는 **생성이 아니라 사전 검색**이라 환각이 없어요. 출처는 표준국어대사전·우리말샘." | 2번 RAG terms(faithfulness=1.0, 출처 표기) |
| 7 | 탭 닫기 → 결과 화면 | "세션이 끝나면 최종 점수와 **케어 전/후** 성장 그래프가 나옵니다." | `GET /session/result` → literacy 52.2, comprehension/engagement/난이도 근거 |
| 8 | 결과의 근거·검증 배지 | "이 점수는 **왜 그런지 근거가 있고, 항상 같은 입력이면 같은 값**입니다. 시스템이 스스로 결과를 검증(warnings)하고, QA 리포트도 함께 남깁니다." | score_breakdown + self-correction warnings + `qaEvaluation`(passed) |

**클로징(5초)**: 0번 한 문장 반복 → "요약 도구가 아니라, 읽는 힘을 지키는 케어입니다."

> 무설치 심사 대비: 확장 로드가 어려우면 **웹 읽기 화면(무설치 폴백)** 또는 **결정론 데모**(`python -m pytest backend/app/tests/test_m1_demo_smoke.py`, 항상 같은 결과)로 대체.

## 2. 심사 방어 Q&A (예상 질문 + 정직한 답)

### Q1. "ChatGPT 요약이랑 뭐가 다른가요?"
- 요약 도구는 **글을 대신 읽어 사람의 읽기를 없앱니다.** 우리는 반대로 **사람이 읽는 과정을 측정·개입·평가**해서 읽는 힘을 **회복**시킵니다.
- 산출물이 "요약문"이 아니라 **읽기 행동 로그 + 개입 기록 + 설명 가능한 점수 + 성장 추세**입니다. 챗봇 한 개가 아니라 문서분석·행동분석·개입판단·점수화·성장관리를 **여러 에이전트가 나눠** 하나의 세션 상태를 공유합니다.

### Q2. "점수를 AI가 매기면 재현·신뢰가 안 되지 않나요?" ★핵심
- **점수는 LLM에 맡기지 않습니다.** `score.py`의 **순수 함수**로 계산합니다:
  `literacy = 이해도×0.5 + 집중도×0.35 + 난이도×0.15 − 비정상읽기감점`, 0~100 clamp, NaN 방어.
- **같은 입력이면 항상 같은 점수**(pytest로 박제), 그리고 `score_breakdown`으로 **왜 그 값인지** 근거를 남깁니다. LLM은 **재구성·용어풀이에만** 쓰고 **점수 계산엔 안 들어갑니다** → 재현성 보장.

### Q3. "용어 뜻이 틀리면(환각) 어쩌죠?"
- 용어풀이는 **생성이 아니라 신뢰 사전 검색**입니다(표준국어대사전·우리말샘·TTA). 실제 데모에서 매칭된 용어의 출처가 사전이고 faithfulness=1.0입니다.
- (정직) faithfulness가 현재 **"사전 직접 인용이므로 1.0"인 설계상 상수**임은 QA 리포트에 분리 표기합니다. 생성 유추가 섞이는 hover 폴백은 "AI 참고용" 라벨로 구분합니다.

### Q4. "집중도를 어떻게 아나요? 정확한가요?"
- 카메라·센서가 아니라 **읽기 행동 신호**(스크롤 속도, 탭 이탈 blur, 멈춤 pause)로 추정하는 **투명한 휴리스틱**입니다. 블랙박스가 아니라 규칙이 공개돼 있어 설명·조정이 됩니다.
- 개입 임계(75/50/30)와 감점 규칙이 문서화돼 있고, 회복하면 개입이 사라져 **과잉 개입을 방지**합니다.

### Q5. "실제로 통합돼서 돌아가나요, 목업 아닌가요?"
- **team/main에 통합**돼 있고, 코어 pytest **100 passed**, 7/8 **실브라우저(Playwright) 왕복 E2E**로 개입 단계 상승과 오버레이 렌더까지 확인했습니다.
- 2번(콘텐츠·RAG)·3번(집중도)·5번(QA) 실모듈이 오케스트레이터에 **실연결**돼 있습니다(stub↔real 토글로 안전 폴백 유지).

### Q6. "데모 중 백엔드/모듈이 죽으면요?"
- 어느 에이전트가 실패해도 **중립값 fallback + trace 기록**으로 흐름이 유지됩니다(데모가 안 끊기게 설계). 확장은 백엔드 불가 시에도 죽지 않고, 결정론 스모크 데모로 대체 가능합니다.
- 시스템이 **스스로 결과를 검증**(self-correction `warnings`)해 빈 출력·비정상 점수를 표면화합니다.

### Q7. "개인정보·프라이버시는요?"
- **로그인 없음**, 설치별 **익명 UUID**만 사용(PII 없음). 로컬 PDF는 서버 업로드 없이 브라우저에서 직접 렌더. 오버레이는 **Shadow DOM 격리**라 페이지와 충돌하지 않습니다.

### Q8. "어떤 LLM을 쓰나요?" (기획서 정정 필요)
- 재구성·용어 유추는 **Gemini(무료 티어, SnowChat 게이트웨이)**를 씁니다. (기획서 초안의 "Claude"는 **비용 0 원칙**에 맞춰 Gemini로 전환됐습니다 — 발표 시 정정.)
- **점수는 어떤 LLM에도 의존하지 않습니다**(Q2).

### Q9. "확장성·실용성은?"
- 브라우저에서 글 읽는 **모든 순간**에 동작(별도 습관 불필요). 웹·PDF를 **동일한 공용 모듈**로 처리해 표면이 늘어도 유지비가 낮습니다. 익명 프로필로 반복 사용 시 성장 추적이 누적됩니다(영구 저장은 DB 레이어).

## 3. 먼저 밝힐 것 (정직 원칙 — 기획서 §4.3 정신)

숨기지 말고 **선제적으로** 한 줄씩 말하면 오히려 신뢰를 얻는다:
- **PDF 라이브 추출**은 실확장 로드가 필요해, 심사 현장에선 확장 로드 후 시연하거나 웹 경로로 대체.
- **크로스세션 프로필 누적**은 영속 저장소(DB) 몫 — 현재 코어는 식별·계약까지 검증, 누적은 DB 연동 후.
- **QA relevance 지표**는 무료 휴리스틱(단어 겹침) — Ragas 교체는 후속. `verified/unverified` 분리 표기.
- **일부 UI**(성장 대시보드·단어 툴팁 상호작용)는 프론트(4번) 산출물 — 백엔드 계약은 제공 완료.
